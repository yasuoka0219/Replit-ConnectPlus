【重要：絶対に守るルール】

- 既存機能を壊さないことを最優先とします。
- 既存のAPI・クエリ・モデル・コンポーネント名は削除・改名をしない。
- 既存のファイル・フォルダ構造を変更しない。
- 可能な限り既存ロジックを再利用し、重複実装を避ける。
- 削除するコンポーネントは「コード上では残す」「UIから非表示にする」など、安全な方法で対処すること。
- 仕様が曖昧な場合は、現状の動きを維持しつつ、新しい構成へ安全に寄せる。

---------------------------------------------
【目的】

ダッシュボードを「最適な営業意思決定ができる構成」へアップグレードする。

以下の３点を実現する：

1. **KPIの意味が分かりやすく、行動につながる UI にする**
2. **重複データを整理し、勝ち筋が直感的に理解できる構成にする**
3. **深掘り分析（クロス集計）とスナップショット（ダッシュボード）を明確に分離する**

既存機能は壊さず、必要があれば既存項目を UI 上から削除してよい。

---------------------------------------------
【実装内容：ダッシュボードの最適化】

### ① KPIカードの再設計（最上段）

以下4つに並べ替えて統一する：

- 今月の売上（前月比 % を追加して表示）
- パイプライン総額（前月比 % または目標比）
- 成約率（前月比 %）
- 新規リード数（前月比 %）

※ 既存APIで前月データが取れる場合は流用し、無い場合は「差分計算用のヘルパー関数」を追加する形で実装。  
※ 色は既存のカード色を踏襲しつつ、彩度をやや落として統一感を出す。

### ② “サブKPI（企業数・連絡先数・案件数・総額）” を1段下にまとめる

現在バラバラに並んでいる以下のカードを、  
**「サマリー」セクションとして1行にコンパクトに配置**：

- 企業数  
- 連絡先  
- 案件数  
- 総額（累計売上）

※ カードサイズは小さく、メイントップのKPIより優先度が低いことが視覚で分かるようにする。

### ③ ステージ別グラフの統一

現在のダッシュボードでは **ステージ別棒グラフ**があり、  
クロス集計にも **ステージ別ファネル**があるため重複している。

ダッシュボードでは **件数のみのシンプルなファネル** に統一し、  
金額含むリッチなファネルはクロス集計ページに保持する。

実装内容：

- ダッシュボード → “ステージ別件数のみファネル”
- クロス集計 → “件数＋金額の本格ファネル”

### ④ 期間別推移を（売上+新規顧客+受注率）の3軸構成のまま維持しダッシュボードに配置

改善ポイント：

- 線の色を統一し、視認性を向上
- 受注率（%）は右軸固定
- 表示期間の初期値を「直近3ヶ月」 or 「直近6ヶ月」に設定

### ⑤ 勝ち筋サマリー（重要！）

現在の「リードソース別」「業界別」1行表示を、  
**トップ3まで表示できるミニテーブル** に拡張する。

- 左：リードソース別（受注率・単価の上位3）
- 右：業界別（売上 or 受注率の上位3）
- 両方に「すべて見る →」リンクを配置（クロス集計ページへ遷移）

※ データ取得ロジックはクロス集計ページの既存の関数を利用し、  
　ソート後上位3件だけ抽出する。

### ⑥ 最近の案件・最近のタスクを最下段に移動（レイアウト維持）

変更なし、ただし以下改善：

- タスクを「今日」「今週」などでフィルタリング可能にする準備だけ記述（実装は任意）
- 案件カードのステータス表示（OPEN, WON, LOST）色を統一

---------------------------------------------
【クロス集計ページの最適化】

以下の順番に並び替え：

1. リードソース別（最重要）
2. 業界別
3. 担当者別
4. ステージ別ファネル
5. 期間別推移
6. 失注理由分析

理由：意思決定 impact の高い順に並べる。

すべて既存のコンポーネントを流用し、順序だけを変更する。

---------------------------------------------
【業界別分析ページの改善】

- 上部フィルタ：業界 / 期間 / 分析軸（受注率・売上・単価・リードタイム）
- 中央：分析軸に応じてグラフ切り替え
- 下部：業界別の案件サマリー（クロス集計のロジック再利用）

---------------------------------------------
【削除して良い要素】

以下の項目は役割が重複するため UI から削除してよい：

- 「案件ステージ別」棒グラフ（ダッシュボード側）
- 「業界別受注率ランキング」（ダッシュボード側）

コードは残しても良いが、UI表示は OFF。

---------------------------------------------
【UIデザインの統一方針】

- KPIカードの色味を少し低彩度に調整
- セクションタイトルとサブタイトルは統一フォーマット
- 余白・カード間隔・フォントサイズをページ全体で統一
- グラフの色も統一テーマに寄せる（青系・紫系メイン）

---------------------------------------------
【最後に】

以上の変更点を、既存コードを壊さず、安全な差分として実装してください。  
必要であれば、内部ヘルパーや selector 関数を追加しても構いません。  
既存のクエリやモデルを大幅に変更する必要はありません。

---------------------------------------------
以上を踏まえて、ダッシュボード・クロス集計・業界別分析の構造を “最適な営業意思決定ができるUI” へ改善してください。
